## 2. AAUM Data Pipeline Fixes

### Problem
Fund AAUM values were wrong — plan-options (Direct Growth, Direct IDCW, Regular Growth, Regular IDCW) were not being summed together per scheme.

### Fixes in `src/data/03_load_aum.py`
- **Removed Direct Plan filter** — now ALL plan-options are included and summed
- **Rewrote `extract_base_fund_name`** with comprehensive regex patterns:
  - Handles `"Fund - Direct Plan - Growth"` and `"Fund-Direct Plan-GROWTH OPTION"`
  - Handles `"FUND DIRECT GROWTH"` (no "Plan" keyword)
  - Handles `"- Daily IDCW Option"`, `"- Weekly IDCW"`, `"- Retail Plan"`, `"- Fortnightly IDCW"`
  - Handles `"- Standard Reinvestment of Income Distribution..."` long suffixes
  - Handles `"- Direct - IDCW"` (dash between Direct and IDCW)
- **Case-insensitive groupby** — `"SBI EQUITY HYBRID FUND"` and `"SBI Equity Hybrid Fund"` now merge
- **Whitespace normalization** — `"Kotak  Arbitrage"` (double space) collapses to `"Kotak Arbitrage"`
- **Result**: 0 duplicate funds, 0 mismatches between raw Excel and parsed output

### Fixes in `src/data/04_merge_master.py`
- Added `processor=rfuzz_utils.default_process` (case-insensitive) to all fuzzy matching
- AAUM match rate: 76.4% -> 99.2% (1916/1931 funds)
- Brokerage match rate: 86.0% -> 87.3%

### AUM -> AAUM Rename
- All user-facing labels changed from "AUM" to "AAUM" across the dashboard
- **Exception**: Client Insights page keeps "AUM" (refers to firm's AUM, not fund's AAUM)
- 56+ label changes across Fund Ranker, Peer Comparison, Portfolio Review, AMC Concentration, Brokerage vs Performance, Recommended Portfolios, Upload page, and AI Chatbot

### Baseline AAUM Auto-Regeneration
- App now auto-detects when `average-aum.xlsx` is newer than the cached CSV
- Automatically re-runs full pipeline (parse -> merge -> score) on startup
- No more stale baseline data

### Session-Only Upload Override
- Upload AAUM Data page no longer writes to disk
- Uploaded AAUM is processed in-memory, stored in `session_state`
- Baseline on disk is never touched — reverts on session close
- "Revert to Baseline" button to clear override mid-session

### Dynamic KPI Cards
- "Total Funds" = funds with AAUM + return data (not fixed count)
- "With Brokerage Data" = funds with AAUM + returns + brokerage
- Both update when new AAUM data is uploaded

## 3. Portfolio Exposure Quantification

### New Feature: Exposure vs Quality Analysis

For each scheme in the portfolio:
- **Exposure %** = scheme AUM / total portfolio AUM
- **Quality %** = composite score percentile within sub-category
- **Exposure Gap** = Exposure % - Quality %

### 4 Categories

| Category | Condition | Action |
|---|---|---|
| **Overexposed** | High weight, low quality | Reduce / shift |
| **Underutilized** | Low weight, high quality | Increase allocation |
| **Well-balanced** | High weight, high quality | Hold |
| **Dead Weight** | Low weight, low quality | Exit |

### Dashboard (Portfolio Exposure Review page)
- 4 color-coded KPI cards with counts and AUM per category
- Interactive scatter plot (X = Quality %, Y = Weight %, sized by AUM)
- 3 tabs with sortable tables: Overexposed, Underutilized, Dead Weight
- Brokerage uplift callout if rebalancing would increase revenue

### Agent Tool
- `quantify_portfolio_exposure` returns the same analysis as text
- Agent's portfolio fallback now uses the default `Scheme_wise_AUM.xls` (same as dashboard page) so results match
